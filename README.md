# stroop-test-web

神経疾患（パーキンソン病など）の症例判別を目的としたストループテストの Web 実装です。
反応時間に加えて、マウス・キーボード・デバイスモーションなどの入力挙動を詳細に計測し、
機械学習モデルの学習データとして出力できます。

将来的に Android 版（`stroop-test-android`）を作成し、同一の計測仕様でモバイル実機でも
データを取得できるようにする予定です。

## 技術スタック

- React 18 + TypeScript
- Vite（開発サーバー / ビルドツール）
- CSS（外部ライブラリなし）

## セットアップ

```bash
npm install
npm run dev
```

開発サーバーが http://localhost:5173/ で起動します。

## 画面フロー

1. **被験者情報入力** — ID（自動生成）、年齢、性別、利き手、備考
2. **ストループテスト** — 全 20 試行（一致条件・不一致条件が 50:50）
3. **結果表示** — 集計結果の表示と計測データのエクスポート

### 回答方法

- 画面の色ボタンをクリック
- またはキーボードの数字キー `1` 〜 `4` で対応する色を選択

## 計測しているデータ

| カテゴリ | 項目 | 想定される臨床的意義 |
|---|---|---|
| 反応時間 | 刺激表示から回答までの時間 (ms) | 運動緩慢、注意機能 |
| マウス軌跡 | 座標・速度・加速度の時系列 | 振戦、運動制御 |
| マウスジッター | 連続点の方向変化量の平均 | パーキンソン振戦の指標 |
| マウス移動距離 | 試行あたりの総移動距離 | 運動効率 |
| クリック精度 | 目標中心からのズレ | 運動精度 |
| クリック押下時間 | mousedown 〜 mouseup の時間 | 固縮、意図しない長押し |
| キー押下時間 | 各キーの dwell time | タイピングリズム |
| デバイスモーション | 加速度（x, y, z） | モバイル実行時の手の振戦 |
| ストループ効果 | 不一致条件の RT − 一致条件の RT | 実行機能・干渉制御 |

## エクスポート形式

結果画面から以下の 3 種類のファイルをダウンロードできます。

- **試行サマリー CSV** — 1 試行 1 行。機械学習用の特徴量として利用想定
- **マウス軌跡 CSV** — 全試行のマウス座標の時系列（詳細分析用）
- **全データ JSON** — セッション全体の生データ（再解析・長期保存用）

ファイル名には被験者 ID と開始日時が自動付与されます。

## プロジェクト構成

```
src/
├── types/index.ts              # 計測データの型定義
├── hooks/
│   ├── useTimer.ts             # 高精度タイマー（performance.now ベース）
│   └── useInputTracking.ts     # マウス・キーボード・モーション計測
├── components/
│   ├── ParticipantForm.tsx     # 被験者情報入力
│   ├── StroopTask.tsx          # ストループテスト本体
│   └── Results.tsx             # 結果表示・エクスポート
├── utils/exportData.ts         # CSV / JSON 出力
├── App.tsx                     # 画面遷移の管理
├── App.css                     # スタイル
├── index.css                   # グローバルスタイル
└── main.tsx                    # エントリーポイント
```

## 試行数・条件の変更

`src/App.tsx` 冒頭の定数を編集してください。

```ts
const TOTAL_TRIALS = 20;      // 試行数
const CONGRUENT_RATIO = 0.5;  // 一致条件の割合（0〜1）
```

## 色の追加・変更

`src/types/index.ts` の `COLORS` 配列を編集することで、刺激色を追加・変更できます。
色を増やすと、`StroopTask.tsx` の数字キー割り当ても自動的に拡張されます。

## 備考

- 現状はローカル動作のみを想定しています（サーバーへの送信なし）。
- 被験者の個人情報は保存しません。任意で入力する備考欄に症例名などを記録できます。
- Android 版でも同じデータ構造を採用する予定のため、本リポジトリの `types/index.ts`
  を共通仕様のリファレンスとして扱います。
